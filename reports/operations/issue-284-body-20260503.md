# 정원 첫 화면을 생산 엔진 중심으로 재배치해 수확·납품을 한 장면에 묶는다

## 요약

#275/#282로 달빛 resident stage와 돌보기 보상은 복구됐지만, 모바일 첫 정원 화면은 여전히 playfield, resident, 자동 생산, 주문 납품, 다음 성장 선택이 별도 카드 스택으로 읽히는 순간이 많다. 이번 WorkUnit은 첫 화면을 더 production game scene처럼 재배치해 `돌보기`, `생산 잎 수령`, `주문 납품`, 다음 목표가 한 장면의 HUD/플레이필드 흐름으로 읽히게 만든다.

## Studio Campaign Gate

- Source of truth: GitHub issue/PR/GateEvent. local campaign ledger 또는 stash는 작업 권한이 아니다.
- Campaign: `P0.5 Idle Core + Creative Rescue`.
- Production gap: 경쟁작 idle game은 첫 화면에서 worker, resource flow, crate/progress, primary claim/deliver verb가 동시에 보인다. 현재 화면은 기능은 풍부하지만 action surface가 카드/설명 stack으로 읽혀 production engine clarity가 흔들린다.

## 후보 비교

1. **선택 / 큰 방향 점프: 첫 화면 production engine layout 재배치**
   - player verb: `돌보기`, `생산 잎 수령`, `주문 납품`, `다음 목표 보기`.
   - production/progression role: 자동 생산, 주문/납품, 다음 성장 선택을 한 screen moment로 묶는다.
   - screen moment: 달방울 누누 발견 후 정원 첫 화면과 복귀 후 30초.
   - visual payoff: playfield state 보호, compact HUD affordance, order crate visual state, reward/progress motion 위치 정리.
   - playtest evidence: Browser Use `iab` 우선, blocker 시 current-session blocker + Playwright screenshot/metrics.
2. **asset/FX 후보: 달방울 누누 idle/care sprite strip 제작**
   - 장점: 귀여움과 감정표현이 커진다.
   - 보류 사유: 신규 game graphics asset은 gpt-image-2/Codex native generation provenance와 review/manifest까지 필요하다. 현재 더 큰 blocker는 이미 있는 actor/production UI가 한 장면의 engine으로 읽히지 않는 점이다.
3. **작은 연결 후보: 도감 기억 도장에서 다음 씨앗/원정 CTA로 이동**
   - 장점: #282 후속으로 자연스럽다.
   - 보류 사유: 직전 issue 인접 기능이며, Production Bar의 핵심인 first-screen production readability를 크게 바꾸지 못한다.
