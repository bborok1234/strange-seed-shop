# 정원에서 달방울 누누를 돌보면 도감 기억 보상이 움직인다

## 요약

#275로 복구한 달빛 resident stage가 아직 `setCreatureStageReaction` 로컬 반응에 머무르므로, `돌보기`를 save-backed 진행/보상 순간으로 연결한다. 목표는 정원 첫 화면에서 달방울 누누를 돌보면 작은 기억 도장/보상 motion이 보이고, 도감의 큰 memory photo에 그 돌봄 기록이 남는 production vertical slice다.

## Studio Campaign Gate

- Source of truth: GitHub issue/PR/GateEvent. local campaign ledger 또는 stash는 작업 권한으로 쓰지 않는다.
- Campaign: `P0.5 Idle Core + Creative Rescue`.
- Production gap: #275의 creature stage는 귀여운 장면을 만들었지만, 돌보기 player verb가 progression/reward로 저장되지 않아 경쟁작 idle game 수준의 “행동 → 보상 → 기록” 루프가 약하다.

## 후보 비교

1. **선택: 달빛 resident 돌보기 → 기억 도장 → 도감 보상 motion**
   - player verb: 정원 stage의 `돌보기`/`단서 따라가기`.
   - progression role: save-backed care memory 또는 one-time reward claim으로 도감 기록/보상 상태가 남는다.
   - screen moment: 달방울 누누 발견 후 첫 정원 화면과 도감 탭 상단 memory photo.
   - visual payoff: reward motion / HUD affordance / album stamp. 새 manifest asset 없이 CSS/DOM motion만 사용한다.
   - playtest evidence: Browser Use `iab` 우선, blocker 시 current-session blocker + Playwright fallback screenshot.
2. **큰 방향 점프 후보: 첫 화면 전체를 card stack에서 full playfield-first layout으로 재배치**
   - 장점: Production Bar에 가장 큰 체감 변화를 줄 수 있다.
   - 보류 사유: #275로 방금 resident stage를 복구했으므로, 전체 레이아웃 재배치 전에 stage의 핵심 동사와 reward loop가 실제로 작동해야 한다. 지금 바로 재배치하면 검증 대상이 너무 넓어지고 Browser Use blocker가 있는 상황에서 시각 회귀 위험이 크다.
3. **asset/FX 후보: 달방울 누누 idle/care sprite strip 신규 제작**
   - 장점: 감정 표현이 가장 직접적으로 좋아진다.
   - 보류 사유: 신규 game graphics asset은 `gpt-game-asset-plan -> prompt -> generate -> review` 또는 gpt-image-2 provenance가 필요하고, 현재 핵심 gap은 새 그림보다 player verb가 저장/보상으로 이어지지 않는 것이다. 이번 WorkUnit은 reward motion으로 game-feel payoff를 만들고, 신규 asset은 후속 후보로 둔다.

## Strategic Jump Check

선택 후보는 직전 issue의 작은 연결 기능처럼 보일 수 있지만, 실제 blocker는 “보이는 캐릭터가 플레이어 행동/진행을 만들지 못함”이다. 이는 첫 5분 fun rubric의 `attachment`와 `first_5m_clarity`를 동시에 막는다. 큰 레이아웃 점프나 신규 asset 제작보다 먼저 이 동사-보상 루프를 고정해야 이후 art/레이아웃 투자 효과를 검증할 수 있다.
